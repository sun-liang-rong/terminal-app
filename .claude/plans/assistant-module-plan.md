# 助手功能模块需求文档

## 1. 功能概述

助手模块是一个命令查询工具，帮助用户快速找到忘记的终端命令。用户只需描述使用场景，即可获取相关命令并一键复制。

## 2. 用户场景

### 2.1 典型使用流程
1. 用户在终端操作时忘记某个命令
2. 点击左侧菜单"助手"进入助手面板
3. 在输入框描述场景，如："查看端口占用"、"解压zip文件"、"查找大文件"
4. 系统展示匹配的命令列表
5. 用户点击复制按钮，命令复制到剪贴板
6. 切回终端粘贴使用

### 2.2 示例查询
| 用户输入 | 返回命令 |
|---------|---------|
| 查看端口占用 | `netstat -tulnp` / `lsof -i :端口号` |
| 解压tar文件 | `tar -xvf file.tar` |
| 查找大文件 | `find . -size +100M` |
| 查看磁盘空间 | `df -h` |
| 杀死进程 | `kill -9 PID` |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 搜索输入
- 输入框支持自然语言描述
- 支持关键词搜索（如 "port"、"文件"、"进程"）
- 实时搜索，输入即显示结果
- 支持清空输入

#### 3.1.2 命令展示
- 展示命令内容
- 展示命令说明/用途
- 展示参数说明（可选）
- 区分不同操作系统（Windows/macOS/Linux）

#### 3.1.3 一键复制
- 每条命令旁有复制按钮
- 点击后显示"已复制"反馈
- 自动复制到系统剪贴板

#### 3.1.4 命令分类（可选）
- 文件操作
- 进程管理
- 网络相关
- 系统信息
- Git操作
- Docker操作
- 其他

### 3.2 交互设计

#### 3.2.1 空状态
- 显示热门/常用命令推荐
- 显示分类标签快速筛选

#### 3.2.2 搜索状态
- 输入时实时过滤
- 无匹配结果时显示友好提示
- 建议相关关键词

#### 3.2.3 复制反馈
- 按钮图标变化或颜色变化
- 短暂显示"已复制"文字
- 1.5秒后恢复原状

## 4. 技术方案

### 4.1 数据来源（二选一）

#### 方案A：预设命令库
- 内置常见命令数据库（JSON文件）
- 本地搜索匹配
- 优点：快速、离线可用
- 缺点：命令数量有限

#### 方案B：AI生成（推荐）
- 调用AI接口生成命令
- 支持任意自然语言查询
- 优点：灵活、覆盖面广
- 缺点：需要网络、有延迟

### 4.2 界面结构

```
┌─────────────────────────────────────┐
│  🔍 搜索命令...                      │  ← 搜索输入框
├─────────────────────────────────────┤
│  热门标签: [端口] [文件] [进程] ...  │  ← 快捷标签
├─────────────────────────────────────┤
│  ┌─────────────────────────────────┐│
│  │ 查看端口占用                     ││  ← 命令卡片
│  │ $ lsof -i :8080        [复制]   ││
│  │ 查看指定端口的占用进程           ││
│  └─────────────────────────────────┘│
│  ┌─────────────────────────────────┐│
│  │ 查找大文件                       ││  ← 命令卡片
│  │ $ find . -size +100M   [复制]   ││
│  │ 查找当前目录下大于100M的文件     ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 4.3 数据结构

```typescript
interface Command {
  id: string
  name: string           // 命令名称
  command: string        // 命令内容
  description: string    // 命令说明
  category: string       // 分类
  tags: string[]         // 搜索标签
  platform?: 'all' | 'windows' | 'macos' | 'linux'
}
```
